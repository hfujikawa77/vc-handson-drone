# ドローンWeb制御アプリケーション 構築用プロンプト

以下の指示に従って、ドローンをWebブラウザから操作できる最小構成のアプリケーションを新規実装してください。

## あなたの役割

あなたは、Python とブラウザフロントエンドに強いシニアエンジニアです。  
FastAPI、WebSocket、pymavlink、HTML、JavaScript、CSS を使い、ローカルで動作するドローンWeb制御アプリケーションを構築してください。

## プロジェクトの目的

既存の CLI ベースのドローン制御体験を、Web ブラウザから操作できるように置き換えることが目的です。  
ユーザーはブラウザ上でドローンへ接続し、状態をリアルタイムに確認しながら、アーム、離陸、着陸、モード変更、指定座標への移動を実行できる必要があります。

## 技術制約

- バックエンドは Python を使うこと
- Web サーバーは FastAPI を使うこと
- リアルタイム通信は WebSocket を使うこと
- MAVLink 通信は `pymavlink` を使うこと
- フロントエンドは素の `HTML + JavaScript + CSS` で実装すること
- 地図表示は Leaflet と OpenStreetMap タイルを使うこと
- フロントエンドのビルドツールは不要
- 単一リポジトリ内に `backend` と `frontend` を分けること

## 実装すべき主要機能

### 1. ドローン接続

- 接続先はデフォルトで `tcp:127.0.0.1:5762`
- ブラウザから「接続」操作を行うと、バックエンドがドローンへの接続を試みる
- 接続成功後、接続状態を保持する
- 接続状態をフロントエンドに表示する

### 2. ドローン制御

以下のコマンドを Web UI から送信可能にすること。

- 接続
- アーム
- 離陸
- 着陸
- GoTo
- モード変更

それぞれの挙動は次の通りです。

- アーム実行前に `GUIDED` モードへ切り替えを試みる
- 離陸実行前に `GUIDED` モードへ切り替えを試みる
- GoTo 実行前に `GUIDED` モードへ切り替えを試みる
- 離陸では目標高度を入力できること
- GoTo では緯度、経度、高度を入力できること
- モード変更では `GUIDED`、`AUTO`、`RTL`、`LOITER`、`STABILIZE` を選択できること

### 3. リアルタイム状態表示

MAVLink メッセージを継続的に受信し、以下をリアルタイム更新すること。

- 接続状態
- アーム状態
- フライトモード
- 緯度
- 経度
- 高度
- ヘディング

状態管理用のデータ構造は最低限以下を持つこと。

```json
{
  "connected": false,
  "armed": false,
  "mode": "UNKNOWN",
  "latitude": 0.0,
  "longitude": 0.0,
  "altitude": 0.0,
  "heading": 0
}
```

### 4. 地図表示

- Leaflet で地図を表示すること
- 初期中心座標は東京駅付近 `35.681236, 139.767125` でよい
- ドローン位置をマーカーで表示すること
- 位置更新のたびにマーカー位置を更新すること
- 飛行軌跡をポリラインで表示すること
- WebSocket 再接続時には過去の飛行軌跡をクリアすること
- 新しい位置を受信したら地図中心をその位置へ寄せること

## バックエンド要件

### 使用ライブラリ

`backend/requirements.txt` には最低限以下を含めること。

```txt
fastapi
uvicorn
websockets
pymavlink
```

### FastAPI の責務

- `/static` に `frontend` ディレクトリをマウントする
- `GET /` で `frontend/index.html` を返す
- `WebSocket /ws` を提供する

### WebSocket の責務

- 接続時に現在のドローン状態を JSON で即時送信する
- クライアントから JSON コマンドを受信する
- サポートする `type` は以下
  - `connect`
  - `arm`
  - `takeoff`
  - `land`
  - `goto`
  - `mode`
- コマンド受信後は、操作結果メッセージを `{"type":"status","message":"..."}` 形式で返すこと
- テレメトリー更新時は状態オブジェクトそのものを送ること

### MAVLink 処理

- `mavutil.mavlink_connection(connection_string, wait_heartbeat=True)` を使って接続すること
- 接続後に `wait_heartbeat()` を呼び出すこと
- 接続成功後に位置系データストリーム要求を送ること
- `recv_match(blocking=True, timeout=0.1)` を使ってメッセージを読むこと
- WebSocket イベントループをブロックしないよう、受信処理は executor 経由で回すこと

### メッセージ処理

- `GLOBAL_POSITION_INT` 受信時に緯度、経度、高度、ヘディングを更新すること
- `HEARTBEAT` 受信時にアーム状態とモードを更新すること
- モード名は `vehicle.mode_mapping()` を使って逆引きすること

### 各コマンドの実装方針

- `connect`: 未接続なら接続処理を開始
- `arm`: `MAV_CMD_COMPONENT_ARM_DISARM` を送信
- `takeoff`: `MAV_CMD_NAV_TAKEOFF` を送信
- `land`: `MAV_CMD_NAV_LAND` を送信
- `goto`: `set_position_target_global_int_send` を使って目標位置を送信
- `mode`: `vehicle.set_mode()` を使ってモード変更

- 離陸高度入力 + 離陸ボタン
- GoTo 用の緯度入力

- GoTo 用の高度入力
- GoTo ボタン
- モード選択ドロップダウン
- モード設定ボタン

### WebSocket クライアント仕様

- 接続先は `ws://${window.location.host}/ws`
- ページロード時に自動接続すること
- WebSocket 接続成功時に `{"type":"connect"}` を送ること
- 切断時は 3 秒後に再接続を試みること
- 再接続時に飛行軌跡をクリアすること

### 状態表示の更新仕様

- ステータス値を DOM に反映すること
- 緯度は小数点以下 6 桁で表示
- 経度は小数点以下 6 桁で表示
- 高度は小数点以下 2 桁で表示
- ドローンマーカーのポップアップに緯度、経度、高度を表示すること

## エラーハンドリング方針

- 未接続時に制御コマンドを送っても致命的に落ちないこと
- WebSocket 切断時は受信タスクを安全に停止すること
- MAVLink 受信で例外が出た場合はログを残すこと
- コマンド送信の成否は即断せず、状態更新はテレメトリー受信ベースで反映すること

## 重要な実装ニュアンス

- `connect` はサーバー起動時に自動実行しないこと
- 接続はユーザー操作またはフロントエンド初回接続契機で始めること
- WebSocket 接続直後に初期状態を送ること
- テレメトリーは新規メッセージ受信時のみフロントへ送ること
- `recv_match` の無限ブロックで WebSocket が停止しないようにすること
- 飛行状態の更新とコマンド送信は疎結合にすること

## 受け入れ条件

以下を満たしたら完成です。

1. サーバー起動後に `http://127.0.0.1:8000/` で UI が表示される
2. UI から WebSocket 接続できる
3. 接続操作で MAVLink 接続が始まる
4. アーム、離陸、着陸、GoTo、モード変更の各コマンドを送れる
5. 緯度、経度、高度、モード、アーム状態がリアルタイムに更新される
6. 地図上のマーカーと軌跡が更新される
7. モバイル幅でも UI が破綻しない
